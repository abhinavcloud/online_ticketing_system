# 1) Small verification SQL pack

Run these in `psql` after bootstrap to confirm the schema and seed are aligned with the finalized services.

***

## 1.1 Verify all required tables exist

```sql
SELECT table_name
FROM information_schema.tables
WHERE table_schema = 'public'
  AND table_name IN (
    'locations',
    'venues',
    'performers',
    'events',
    'event_performers',
    'event_categories',
    'seats',
    'reservations',
    'reservation_seats',
    'tickets'
  )
ORDER BY table_name;
```

### Expected

You should see all 10 tables listed.

***

## 1.2 Verify critical columns used by finalized services

```sql
SELECT table_name, column_name, data_type
FROM information_schema.columns
WHERE table_schema = 'public'
  AND (
    (table_name = 'venues' AND column_name IN ('id','location_id','name','address'))
    OR
    (table_name = 'events' AND column_name IN ('id','name','description','venue_id','event_date','event_time','event_type','status'))
    OR
    (table_name = 'event_categories' AND column_name IN ('id','event_id','name','price','currency'))
    OR
    (table_name = 'seats' AND column_name IN ('id','event_id','category_id','seat_label','status','updated_at','booked_at'))
    OR
    (table_name = 'reservations' AND column_name IN ('id','user_id','event_id','category_id','status','idempotency_key','expires_at','failure_reason'))
    OR
    (table_name = 'reservation_seats' AND column_name IN ('reservation_id','event_id','seat_id','status'))
    OR
    (table_name = 'tickets' AND column_name IN (
      'id','user_id','reservation_id','payment_id','event_id','category_id',
      'venue_id','seat_label','unit_price','total_amount','currency',
      'event_name','event_date','event_time','venue_name','venue_address',
      'category_name','status'
    ))
  )
ORDER BY table_name, column_name;
```

***

## 1.3 Verify category pricing + currency exist

```sql
SELECT id, event_id, name, price, currency
FROM public.event_categories
ORDER BY name;
```

### Expected

Rows like:

* VIP / 3000 / INR
* General / 500 / INR

***

## 1.4 Verify seat inventory exists and status defaults are correct

```sql
SELECT category_id, status, COUNT(*) AS seat_count
FROM public.seats
GROUP BY category_id, status
ORDER BY category_id, status;
```

### Expected

You should see seats under each category with `status = AVAILABLE`.

***

## 1.5 Verify event + venue + category join (used by booking confirmation)

```sql
SELECT
  e.id AS event_id,
  e.name AS event_name,
  e.event_date,
  e.event_time,
  e.event_type,
  v.id AS venue_id,
  v.name AS venue_name,
  v.address AS venue_address,
  ec.id AS category_id,
  ec.name AS category_name,
  ec.price,
  ec.currency
FROM public.events e
JOIN public.venues v ON v.id = e.venue_id
JOIN public.event_categories ec ON ec.event_id = e.id
ORDER BY e.name, ec.name;
```

### Expected

One row per event-category combination.

***

## 1.6 Verify performer mapping (used by browse `/performers` and `/events`)

```sql
SELECT
  e.name AS event_name,
  p.name AS performer_name
FROM public.event_performers ep
JOIN public.events e ON e.id = ep.event_id
JOIN public.performers p ON p.id = ep.performer_id
ORDER BY e.name, p.name;
```

***

## 1.7 Verify tickets table is empty before booking

```sql
SELECT COUNT(*) AS ticket_count
FROM public.tickets;
```

### Expected

Initially `0`.

***

## 1.8 Verify reservations audit table (FAILED rows only in current design)

```sql
SELECT id, user_id, event_id, category_id, status, idempotency_key, failure_reason, created_at
FROM public.reservations
ORDER BY created_at DESC
LIMIT 20;
```

### Expected

Initially empty unless you already had failed reservation attempts.

# 3) Suggested end-to-end test order

Use this exact order when testing manually:

1. **Browse**
   * get `eventId`, `categoryId`
2. **Queue**
   * get `bookingToken`
3. **Seat Availability**
   * verify seats show `AVAILABLE/LOCKED/BOOKED`
4. **Reservation**
   * get `reservationId`, `totalAmount`, `expiresAt`
5. **Payment Mock**
   * get `paymentId`, `status=SUCCESS`
6. **Booking Confirmation**
   * get `ticketIds`, DB seats update to `BOOKED`
